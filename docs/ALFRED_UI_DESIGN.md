## 1. Design Philosophy

### 1.1 Core Principles

| Principle | Description | Anti-Pattern |
|-----------|-------------|--------------|
| **Conversation-First** | Everything through natural language | Forms, dropdowns, complex inputs |
| **Goal-Oriented** | Users speak in goals, not commands | Step-by-step wizards |
| **Transparency** | Show what Alfred is doing/thinking | Black-box responses |
| **Proactive Surface** | Alfred initiates, not just responds | Passive waiting |
| **Minimal Cognitive Load** | One focus at a time | Dashboard overload |
| **Butler Presence** | Alfred feels like a person, not software | Generic AI chat |

### 1.2 Design Mantras

```
"If it needs a tutorial, it's too complex."
"Show intent, not just data."
"Every tap should feel like asking a butler."
"Less chrome, more content."
```

### 1.3 Inspiration Synthesis

**From Manus.ai:**
- Transparent "Alfred's Mind" panel showing reasoning
- Async task execution with progress visibility
- Goal decomposition into visible steps

**From Motion/Reclaim.ai:**
- Auto-scheduling intelligence
- Protected focus time
- Calendar-first view

**From Sunsama/Trevor AI:**
- Minimal, focused interface
- Daily ritual structure
- Clean typography hierarchy

## 3. Core Patterns

### 3.1 The Alfred Presence

Alfred should feel like a person in the room, not software running.

**Avatar States:**
```
[Idle]      - Subtle breathing animation
[Listening] - Pulsing glow, waveform
[Thinking]  - Orbiting dots, processing
[Speaking]  - Animated waveform
[Working]   - Progress indicator with steps
```

**Personality Cues:**
- Use "Sir" or user's name naturally
- Time-appropriate greetings
- Occasional witty observations
- Empathetic acknowledgments

### 4.3 Information Hierarchy

**Today Screen (Home):**
```
1. Greeting + Quick Status (Alfred speaking)
2. Focus Block (THE most important thing)
3. Proactive Cards (Alfred's suggestions)
4. Upcoming (next 3-4 hours)
5. Habits Due (visual streak indicators)
```

```
1. Quick Add (conversational input)
2. Active Tasks (what's in progress)
3. Due Today (priority sorted)
4. Habits Pending (quick check-off)
5. Project Quick Access
```

## 7. Component Library

### 7.1 Core Components

| Component | Usage |
|-----------|-------|
| `AlfredAvatar` | Animated butler icon with states |
| `ConversationInput` | Universal text/voice input |
| `TaskCard` | Task display with swipe actions |
| `HabitChip` | Compact habit with streak |
| `ProactiveCard` | Alfred suggestion cards |
| `CalendarEvent` | Timeline event block |
| `ProjectBadge` | Mini project indicator |
| `EntityPill` | Person/company chip |
| `StreakBadge` | Fire + number indicator |
| `TransparencyPanel` | Alfred's thinking steps |

### 7.2 Alfred Avatar States

```tsx
<AlfredAvatar
  state="idle" | "listening" | "thinking" | "speaking" | "working"
  size="sm" | "md" | "lg"
  showGlow={true}
/>
```

### 7.3 Conversation Input

```tsx
<ConversationInput
  placeholder="Ask Alfred..."
  onSubmit={handleMessage}
  enableVoice={true}
  suggestions={["Set reminder", "Add task", "Update project"]}
/>
```

### 7.4 Proactive Card

```tsx
<ProactiveCard
  type="warning" | "insight" | "reminder" | "celebration"
  title="RSN - No update in 5d"
  description="Everything okay with this project?"
  actions={[
    { label: "Update", onPress: handleUpdate },
    { label: "Snooze", onPress: handleSnooze },
  ]}
  onDismiss={handleDismiss}
/>
```

## Implementation Notes

### Priority Order

1. **P0 - Core Experience**
   - Today screen with briefing
   - Conversational input everywhere
   - Voice interface (basic)
   - Task + Habit unified view

2. **P1 - Intelligence**
   - Transparency panel
   - Proactive cards
   - Evening review flow
   - Entity views

3. **P2 - Polish**
   - Animations
   - Haptics
   - Offline support
   - Widget

### Tech Stack Alignment

- Use existing React Native + Expo setup
- Leverage react-native-gifted-chat for voice UI
- Add react-native-reanimated for fluid animations
- Consider react-native-gesture-handler for swipe actions
